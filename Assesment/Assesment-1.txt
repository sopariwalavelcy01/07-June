Scenario-1 :
You are a junior developer at a fintech startup building a digital wallet app. The team is
debating between Waterfall and Agile methodology. The project has frequently
changing requirements and the client expects to review working features every two
weeks.

Q : Which development methodology would you recommend for this project, and why?
Identify two specific risks the team would face if they chose the wrong methodology for this
context.

ANS :
Recommended Methodology: Agile

Reason:
Agile is the best choice because the project requirements change frequently, and the client wants to review working features every two weeks. Agile delivers software in short sprints, allows regular feedback, and quickly adapts to changing requirements.

Two Risks of Choosing the Wrong Methodology (Waterfall):

1. Difficulty in Handling Changes
   Waterfall is not flexible. If requirements change during development, it can lead to increased cost, delays, and extra rework.

2. Delayed Client Feedback
   In Waterfall, the client reviews the product only after major development is completed. If the client is not satisfied, significant changes may be required, wasting time and resources.

Scenario-2 :
You are collaborating on a college project website with three teammates. Just before a
client demo, one teammate pushed incomplete code directly to the main branch,
breaking the layout for everyone else on the team.

Q : Describe the Git workflow your team should adopt to prevent this from happening
again. Name at least two Git commands your team should use as part of this workflow and
explain the role of each command.

ANS : 
Recommended Git Workflow:

The team should use a feature branch workflow instead of pushing code directly to the main branch. Each teammate should work on a separate branch, test their changes, and create a Pull Request before merging into the main branch. This allows code review and testing, preventing incomplete or broken code from affecting the main branch.

Git Commands:

1. `git checkout -b feature-name`

   * Creates and switches to a new feature branch where a developer can work without affecting the main branch.

2. `git add .`

   * Stages all modified files before committing the changes.

3. `git commit -m "Added new feature"`

   * Saves the changes with a meaningful commit message.

4. `git push origin feature-name`

   * Uploads the feature branch to GitHub so teammates can review it.

5. `git pull origin main`

   * Downloads the latest changes from the main branch before merging, helping avoid conflicts.

This workflow keeps the `main` branch stable, allows code reviews through Pull Requests, and prevents incomplete code from breaking the project.

Scenario :
You are designing a news portal that displays articles in a 3-column grid on desktop
screens and in a single-column layout on mobile devices. Your team wants to use
Bootstrap's grid system to achieve this without writing any custom media query CSS.

Q : Explain how Bootstrap's grid class system works to produce this responsive layout.
Write the Bootstrap class combination you would apply to the article column div, and explain
what each class in the combination controls.

ANS :
Bootstrap Grid System:

Bootstrap uses a 12-column responsive grid system. By applying different grid classes for different screen sizes, the layout automatically changes based on the device width without writing custom media queries.

Bootstrap Class Combination:

```html
class="col-12 col-md-4"
```

Explanation:

* `col-12`

  * On small screens (mobile), each article takes all 12 columns, so only one article is displayed per row (single-column layout).

* `col-md-4`

  * On medium screens and larger (desktop/tablet), each article takes 4 out of 12 columns. Since 12 ÷ 4 = 3, three articles are displayed in one row (3-column layout).

Example:

```html
<div class="container">
    <div class="row">
        <div class="col-12 col-md-4">Article 1</div>
        <div class="col-12 col-md-4">Article 2</div>
        <div class="col-12 col-md-4">Article 3</div>
    </div>
</div>
```

Result:

* Mobile: 1 article per row.
* Desktop (medium and larger screens): 3 articles per row.

SCENARIO
You are writing a C program to record daily rainfall data for a month (30 days). You
need to store all 30 readings, calculate the monthly average, and then identify which
specific days recorded rainfall above that average.

Q : Explain why an array is the correct data structure for this task compared to using 30
separate variables. Describe the two-step logic your program would follow: first to compute the
average, then to identify the above-average days — and explain why both steps cannot be done
in a single loop.

ANS :
Why an Array is the Correct Data Structure:

An array is the best choice because it stores all 30 rainfall readings in a single variable with indexed elements. It makes it easy to access, process, and loop through the data. Using 30 separate variables would make the program longer, harder to manage, and less efficient.

Two-Step Logic:

Step 1: Calculate the Average

* Store the rainfall for all 30 days in an array.
* Use a loop to add all the rainfall values.
* Divide the total by 30 to get the monthly average.

Step 2: Find Above-Average Days

* Use another loop to compare each day's rainfall with the calculated average.
* Print the day numbers where the rainfall is greater than the average.

Why Both Steps Cannot Be Done in One Loop:

The average is not known until all 30 rainfall values have been added. Therefore, the program must first complete the calculation of the average. Only after the average is available can it correctly compare each day's rainfall and identify the above-average days.

SCENARIO
You are debugging a C program that uses a pointer to traverse a string entered by the
user. The program works correctly when the user types a name, but crashes
immediately when the input field is left empty (the user just presses Enter).

Q : Explain why the program crashes on an empty string input and describe the check
your pointer-based solution must perform before dereferencing the pointer. How does traversing
a string using a pointer differ from traversing it using array index notation, and which approach
makes the empty-input bug easier to catch?

ANS :
Why the Program Crashes:

If the user presses Enter without entering a name, the string is empty (`""`). If the program dereferences a pointer without checking it first, it may access invalid memory or go beyond the end of the string, causing the program to crash.

Check Before Dereferencing the Pointer:

Before using `*ptr`, the program should check that:

* The pointer is not `NULL` (if applicable), and
* The current character is not the null character (`'\0'`).

Example:

```c
if (ptr != NULL && *ptr != '\0')
{
    // Safe to access *ptr
}
```

Pointer Traversal vs Array Index Traversal:

* Pointer Traversal:

  * Uses a pointer (`ptr++`) to move through each character until `'\0'` is reached.

* Array Index Traversal:

  * Uses an index (`str[i]`) to access each character one by one.

Which Approach Makes the Bug Easier to Catch?

Pointer traversal makes the empty-input bug easier to catch because the program can directly check `*ptr == '\0'` before accessing the character. This helps prevent dereferencing an invalid or empty string.

#include <stdio.h>

int main()
{
    float hours[7];
    float total = 0, average;
    float max;
    int maxDay = 1;
    int i, j;

    // Input study hours
    for(i = 0; i < 7; i++)
    {
        do
        {
            printf("Enter study hours for Day %d (0-24): ", i + 1);
            scanf("%f", &hours[i]);

            if(hours[i] < 0 || hours[i] > 24)
            {
                printf("Invalid input! Please enter hours between 0 and 24.\n");
            }

        } while(hours[i] < 0 || hours[i] > 24);

        total += hours[i];
    }

    // Find highest study hours
    max = hours[0];

    for(i = 1; i < 7; i++)
    {
        if(hours[i] > max)
        {
            max = hours[i];
            maxDay = i + 1;
        }
    }

    average = total / 7;

    printf("\nWeekly Study Summary\n");
    printf("Total Study Hours : %.2f\n", total);
    printf("Average Per Day   : %.2f\n", average);
    printf("Highest Study Day : Day %d (%.2f hours)\n", maxDay, max);

    // Visual Bar Chart
    printf("\nStudy Hours Bar Chart:\n");

    for(i = 0; i < 7; i++)
    {
        printf("Day %d: ", i + 1);

        for(j = 0; j < (int)hours[i]; j++)
        {
            printf("*");
        }

        printf("\n");
    }

    return 0;
}

#include <stdio.h>

struct Student
{
    char name[50];
    int rollno;
    float marks;
    char grade;
};

void assignGrade(struct Student *s)
{
    if(s->marks >= 90)
        s->grade = 'A';
    else if(s->marks >= 75)
        s->grade = 'B';
    else if(s->marks >= 60)
        s->grade = 'C';
    else if(s->marks >= 45)
        s->grade = 'D';
    else
        s->grade = 'F';
}

void printTopper(struct Student s[], int n)
{
    int i, top = 0;

    for(i = 1; i < n; i++)
    {
        if(s[i].marks > s[top].marks)
        {
            top = i;
        }
    }

    printf("\nTopper Details\n");
    printf("Name  : %s\n", s[top].name);
    printf("Marks : %.2f\n", s[top].marks);
}

int main()
{
    struct Student s[3];
    int i;

    for(i = 0; i < 3; i++)
    {
        printf("\nEnter Details of Student %d\n", i + 1);

        printf("Name: ");
        scanf(" %[^\n]", s[i].name);

        printf("Roll No: ");
        scanf("%d", &s[i].rollno);

        printf("Marks: ");
        scanf("%f", &s[i].marks);

        assignGrade(&s[i]);
    }

    printf("\n---------------------------------------------\n");
    printf("Name\t\tRoll No\tMarks\tGrade\n");
    printf("---------------------------------------------\n");

    for(i = 0; i < 3; i++)
    {
        printf("%-15s %-8d %-7.2f %c\n",
               s[i].name,
               s[i].rollno,
               s[i].marks,
               s[i].grade);
    }

    printTopper(s, 3);

    return 0;
}

struct Expense
{

    float amount;

{

    {

                    printf("Expense list is full!\n");

                printf("Enter Amount: ");

                if(count == 0)

                    printf("No expenses recorded.\n");
                    break;

                total = 0;

                printf("\n------------------------------\n");
                printf("Category\tAmount\n");
                printf("------------------------------\n");

                {
                    printf("%-15s %.2f\n", exp[i].category, exp[i].amount);
